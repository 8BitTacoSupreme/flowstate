# Quick Task 260710-x5a: Harden `_run_trial` error handling

## Objective

`bench/replicate.py::_run_trial` currently wraps subprocess + JSON parse + score extraction
in one `try/except Exception: return None`, collapsing three failure classes into a silent
`None`. A judge-output **contract violation** (malformed JSON, or a `per_run` row missing its
`score` key) is therefore averaged out of the paired-bootstrap CI as if the trial merely had
no data ‚Äî silently poisoning the milestone-gating "real win vs. not" delta. Decision (locked):
**strict** ‚Äî contract violations propagate/halt; legitimate gaps stay `None`.

<action>
In `bench/replicate.py::_run_trial`, restructure the try/except/finally so that:
- Capture `proc = subprocess.run(cmd, check=False)`; if `proc.returncode != 0`, print a diagnostic
  `[replicate] {label}: compound_eval exited {proc.returncode}` (flush=True) and `return None` (class 1 ‚Äî harness gap).
- Read the output file inside `try: ... except OSError as exc:` ‚Äî on OSError print
  `[replicate] {label}: no/unreadable output ({exc})` (flush=True) and `return None` (class 2 ‚Äî output gap).
- Keep `finally: out.unlink(missing_ok=True)` (and the `os.close(fd)` after mkstemp) exactly as-is.
- Move `json.loads(raw)` and `scores = [r["score"] for r in payload.get("judge", {}).get("per_run", [])]`
  to AFTER the try/finally block, so `json.JSONDecodeError` / `KeyError` / `TypeError` (class 3 ‚Äî contract
  violation) propagate uncaught.
- Keep the final guard `if not scores or any(s is None for s in scores): return None` and the
  `return [float(s) for s in scores]`.
- Update the docstring to state the three-class contract (gaps ‚Üí None; contract violations propagate).
- Narrow the catch: replace the broad `except Exception` with `except OSError` only.
Do NOT change `_run_trial`'s signature, callers, or `bench/close_loop.py`.
</action>
