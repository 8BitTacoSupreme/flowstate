<objective>
Improve the LongMemEval QA reader path (diagnosed bottleneck: oracle QA ~0.48 even with a strong judge → the reader is the gap) and make the openai path fail loud instead of silently scoring 0/100.

Three changes, all confined to `bench/longmemeval_qa.py` + `tests/test_longmemeval_qa.py`:
1. A LongMemEval-tuned reader instruction, wired into the claude reader via `_g._answer(..., instruction=...)`.
2. An optional `--reader-provider {claude,openai}` that answers via `_openai_chat` when openai, with a reader-model auto-upgrade mirroring the judge pattern.
3. An upfront openai canary in `_run_qa` that hard-errors (return 1) on any openai API failure, generalized to fire for judge OR reader openai use.

Purpose: lift reader accuracy on gold context and eliminate the silent-failure class where a model-403 produces a fake 0/100.
Output: modified harness + extended offline tests, default behavior byte-identical.
</objective>
