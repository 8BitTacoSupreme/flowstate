<objective>
Unify memory injection so the same `prior_knowledge` block is built ONCE at orchestrator pipeline start and threaded into every tool adapter (Research, Strategy, GSD). Today each tool independently calls `self.get_memory_context(query)` with a query-specific string, producing slightly different prefixes per call and defeating Anthropic's server-side prompt cache (5-min TTL).

Purpose: Eliminate 3-4 redundant FTS5 searches per run, enable server-side prompt-cache hits across Research -> Strategy -> GSD calls, clarify data flow (memory read once, consumed many times). Aligned with arXiv 2412.15605v1 (Cache-Augmented Generation) without adopting full CAG.

Output: Modified `ToolAdapter` base accepts `prior_knowledge: str | None`; orchestrator computes the block once and passes it through; `get_memory_context()` remains for escape-hatch use; full test suite green at >=80% coverage.
</objective>

```python
class ToolAdapter:
    name: str = "base"

    def __init__(
        self,
        root: Path,
        dry_run: bool = False,
        bridge: ClaudeBridge | None = None,
        memory: MemoryStore | None = None,
    ):
        self.root = root
        self.dry_run = dry_run
        self._bridge = bridge
        self.memory = memory

    def get_memory_context(self, query: str, max_tokens: int = 1500) -> str:
        """Retrieve relevant prior knowledge for prompt injection."""
        if self.memory is None:
            return ""
        return self.memory.get_context(query, max_tokens=max_tokens)
```
