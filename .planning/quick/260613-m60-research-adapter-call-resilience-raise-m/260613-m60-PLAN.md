<objective>
Make ResearchAdapter's per-topic research calls resilient to the dominant transient
failure mode: with WebSearch/WebFetch the model often consumes all turns searching and
never emits a final answer, so `claude --print` exits 1 ("Reached max turns") — a ~40%
measured failure rate that guts the report with `*Research failed:*` placeholders. A
separate real failure is returncode 0 with empty stdout (success=True, output="").

Fix: raise max_turns 3 -> 6 and wrap each per-topic call in a bounded retry (up to 3
attempts) that treats `success and output.strip()` as the only success condition. Retry
reuses the SAME prompt/args to preserve the prior-knowledge prefix and Anthropic prompt
cache. Everything else is byte-for-byte identical.

Purpose: Cut the empty-section failure rate without changing prompts, topic splitting, or
any other adapter.
Output: Updated `flowstate/tools/research.py` (execute per-topic loop only) + new tests.
</objective>
