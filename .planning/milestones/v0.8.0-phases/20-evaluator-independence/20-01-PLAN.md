<objective>
Harden `bench/judge.py` so it (a) refuses to run when the judge configuration is compromised (no judge, or a judge equal to the producer) via a fail-loud guard at config/validation time, and (b) turns a single 0-10 judge score into a defensible multi-judge verdict that keeps the 0-10 numeric signal AND adds a binarized pass-rate with a Wilson confidence interval.

Purpose: IND-01 (fail-loud independence guard + CLI surface) and IND-02 (multi-judge aggregation) both live in `judge.py`; this plan owns every `judge.py` change so no other wave collides on the file.
Output: A validation helper, an argparse `main()` / `python -m bench.judge` CLI, a multi-judge aggregation function, a documented pass-threshold constant, and tests — all additive; existing library contracts (`judge_run`, `summarize`, `JudgeResult`) unchanged.
</objective>
