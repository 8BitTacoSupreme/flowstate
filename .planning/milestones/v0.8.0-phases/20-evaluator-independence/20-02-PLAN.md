<objective>
Wire the shared independence guard (built in 20-01) into the ONE real judged-run chokepoint — `bench/compound_eval.py` — so live runs cannot grade their own producer (D-06, corrected). `compound_eval.py` already owns `--judge-model` (default `None`) and a `do_judge` gate; it gains `--producer-model` and calls `_validate_judges` when judging is active. Because the real replicate/close_loop path reaches this chokepoint by spawning `compound_eval --judge --allow-llm` as a subprocess (via `bench/replicate.py::_run_trial`), that conduit is updated to thread explicit, DISTINCT judge/producer models so the guard protects the default path AND leaves it runnable. Then lock in IND-03 with a test proving `bench/metrics.py`'s `compounding_score` stays the authoritative deterministic scorer and the LLM judge (single- OR multi-judge) is excluded from it.

Purpose: IND-01's guard is only defensible if the caller that actually judges enforces it; RunSnapshot carries no producer field (Phase 19 added tokens/wall-clock only), so the producer model must be passed explicitly at the chokepoint. IND-03 is the milestone-wide integrity invariant made executable under the new multi-judge path.
Output: a guard call + `--producer-model` at the compound_eval chokepoint, distinct judge/producer models threaded through the replicate conduit, and a new exclusion + real-guard-path test module.

close_loop.py is intentionally OUT of scope: it never calls `judge_run`; its real path is covered transitively through compound_eval (D-06).
