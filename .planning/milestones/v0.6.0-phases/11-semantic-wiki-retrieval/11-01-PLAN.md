<objective>
Add per-run SEMANTIC top-k retrieval to the context_prefix WIKI layer (seam (b) from
SEMANTIC_RETRIEVAL_HANDOFF.md). When the opt-in `wiki` layer is active AND an embedder
is available AND a wiki CORPUS DIRECTORY exists, embed the corpus articles, build an
ephemeral in-memory vec0 table, KNN the run's `query`, and inject ONLY the top-k articles
under the existing `## Codebase Wiki` heading. On every degraded path (no embedder,
no sqlite-vec, no corpus dir, retrieval error) fall back to the existing static
single-file `_read_wiki_layer` read, byte-identically. The default (include_layers=None)
path stays entirely untouched.

Purpose: closes the milestone — recovers the proven ~0.82 grounding lift (wikivec arm)
for the production wiki layer while keeping the dep-free default install and the golden
byte-identity contract intact.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| disk → process | Reading wiki article files under `.planning/codebase/wiki/` |
| query/articles → embedder | Run query + article text passed to the embedding provider |
| vectors → ephemeral vec0 | In-memory `:memory:` sqlite-vec table for this run only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering | `sqlite3.enable_load_extension` on the throwaway `:memory:` conn | mitigate | Re-scope the extension loader OFF immediately after `sqlite_vec.load(conn)` via `enable_load_extension(False)` (mirror Phase 9 CR-01); conn is local and closed in a `finally`. |
| T-11-02 | Tampering / Injection | vec0 KNN query | mitigate | All vectors bound as parameters via `sqlite_vec.serialize_float32(...)`; no string interpolation of values into SQL (only the validated integer `dim` formats the table DDL). |
| T-11-03 | Information Disclosure | Article glob path | accept | Glob is constrained to `root / .planning/codebase/wiki/**/*.md`; only the operator's own repo files are read; no traversal of user-supplied paths. |
| T-11-04 | Denial of Service | Retrieval/embed failure mid-run | mitigate | `_semantic_wiki_layer` is wrapped in `try/except Exception: return None` → caller falls back to the static `_read_wiki_layer`; never raises, never blocks the prefix build. |
| T-11-SC | Tampering | npm/pip/cargo installs | n/a | No new package installs in this phase — `sqlite-vec` and the optional `fastembed` extra were both vetted in Phase 9. |
</threat_model>
