## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| project_root (caller-supplied Path) -> generated SBPL/bwrap profile | an attacker-influenced path string is embedded into a kernel-interpreted profile |
| generated macOS profile file on disk | temp .sb file readable while sandbox-exec consumes it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-04 | Elevation of Privilege / Tampering | `project_root` with SBPL/shell metacharacters breaks out of the `(subpath "...")` quoting or bwrap `--bind` arg | mitigate | project_root is a `Path` (not shell-interpolated — argv list, no shell); bwrap args are passed as an argv list, never a shell string (no shell metachar surface). For SBPL, note in the builder docstring that project_root containing a `"` would break the profile; golden test pins the emitted shape and a follow-up note flags path-quoting hardening as a Phase-25 confine-runtime concern (builders are not live this phase). |
| T-23-05 | Tampering | malformed macOS profile silently degrades to allow-all | mitigate | Golden byte-exact test on build_macos_profile locks the `(deny file-write*)` line — a regression that drops it fails the test; the profile always starts allow-default then denies (spike-proven auth-preserving shape). |
| T-23-06 | Information Disclosure | temp .sb profile file left world-readable / never cleaned | accept | Profile contains only paths (no secrets — V8); cleanup/permission-hardening of the temp file is a Phase-25 confine-runtime concern (builders not wired live here). Noted for Phase 25. |
| T-23-SC | Tampering | npm/pip/cargo installs | mitigate | Zero packages installed (stdlib only: tempfile/shutil/pathlib). |
