<objective>
Wire the WR-09 temp-profile cleanup at the bridge (the primary confine call site). `_wrap_macos` writes a temp `.sb` profile with `delete=False` (it must outlive the `wrap()` return so `sandbox-exec -f <path>` can read it), but nothing currently unlinks it — every `confine`-tier macOS `claude --print` call leaks one `.sb` file into the OS temp dir. Add a `try/finally` around the existing `subprocess.run` in `ClaudeBridge.run()` that unlinks the profile path after the child exits, regardless of outcome.

Purpose: Confine cannot ship for real production use (SBX-05) with an unbounded temp-file leak. The `(argv, env)` tuple contract is LOCKED (D-04) — cleanup belongs at the call site, not in a new spawn helper.
Output: A `finally`-guarded unlink in `bridge.py` + a test asserting no leak on the success and error paths.
</objective>
