<objective>
Make the confine tier fail loud (SBX-06, D-01). When `sandbox=confine` is requested but NO confinement is achievable — the platform sandbox binary is genuinely absent (`sandbox-exec` on macOS, `bwrap` on Linux) or the platform is not darwin/linux — `wrap()` must RAISE a clear error with a per-platform install hint instead of silently running the command unconfined. This replaces Phase-23's RUNG-3 observe-fallback and the silent `_find_sandbox_exec` `/usr/bin` guess FOR THE EXPLICIT-CONFINE CASE ONLY. Partial capability still degrades within confinement (no raise), and the default `observe` tier is untouched and still never raises. Also folds in D-04: document (do NOT fix) the WR-2 npm `*_TOKEN` scrub limitation.

Purpose: The guardrail is only trustworthy if a requested confinement that cannot be delivered is loud, not silent. A silent unconfined run under `confine` is a confine-bypass — the exact failure this phase exists to prevent.
Output: `SandboxUnavailableError` (or chosen stdlib-derived exception) + fail-loud dispatch in `sandbox.py`, updated tests, and WR-2 documentation comments.
</objective>
