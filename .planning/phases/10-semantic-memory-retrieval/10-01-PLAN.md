<objective>
Wire semantic KNN retrieval into `MemoryStore.get_context()` (handoff seam (a)) so the most
semantically relevant memories surface when vectors exist, while preserving a byte-identical
FTS5/BM25 fallback when the embedder is absent or KNN fails.

Purpose: The bench proved naive FTS5/BM25 retrieval surfaces the wrong memory (the "raw-memory
FTS5 null"); semantic KNN over `memories_vec` recovers oracle-level grounding. This phase makes
`get_context` — the function that produces the auto-injected `## Prior Knowledge` block — use that
semantic path when it can, and degrade silently to today's behavior when it can't.

Output: A modified `flowstate/memory.py` (`get_context` only) and new offline tests in
`tests/test_memory.py`. Reuses the Phase-9 vec0 store, embedder seam, `_vec_ready`, and
`_row_to_entry` verbatim — builds nothing new in the vector layer.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → get_context(query) | `query` is in-process developer/pipeline text; flows into embedding + a parameterized SQLite KNN read. No external/web input, no auth surface. |
| get_context → memories_vec / memories | Local read-only KNN + row lookup on the existing `memory.db` connection. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering (SQL injection) | KNN query in `_semantic_results` | mitigate | Bind the serialized vector blob and integer k as parameters; never interpolate `query` into SQL (mirrors existing `search()` / `_embed_rowid` parameterization). |
| T-10-02 | Denial of Service | KNN failure / sqlite-vec error propagating to caller | mitigate | Whole semantic path wrapped in `try/except Exception: return None` → get_context falls back to FTS5; the function never raises, so a vec fault cannot break prompt assembly. |
| T-10-03 | Information disclosure | `## Prior Knowledge` block content | accept | Block contents are the developer's own stored memories injected into their own prompt; no new exposure beyond today's FTS5 path. |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | No new packages — reuses Phase-9 `sqlite-vec` (already audited) and the optional `[semantic]` extra; no install task in this phase. |
</threat_model>
