## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .planning/ markdown → harvester | VERIFICATION.md / REVIEW.md are local but may be malformed or arbitrarily large; untrusted-shaped input crosses into the parser |
| gotchas.py → filesystem | GOTCHAS.md write + phases/* glob must stay within `root` |
| caller → capture_gotcha | message text is free-form; feeds regex normalization and SQLite |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Denial of Service | _normalize / harvester regex on large/adversarial markdown | mitigate | Bounded reads (cap file size before parsing); anchored, non-backtracking regex (no nested quantifiers) — ReDoS-safe |
| T-07-02 | Tampering | GOTCHAS.md write + phases/* glob | mitigate | Write only to `root/.planning/GOTCHAS.md`; glob rooted at `root/.planning/phases`; no path from untrusted input is used as a write target |
| T-07-03 | Denial of Service | capture/harvest raising into pipeline | mitigate | Self-contained try/except in capture_gotcha, _rewrite_gotchas_md, harvest_planning_gotchas (Phase-6 WR-01) |
| T-07-04 | Tampering | malformed frontmatter / binary VERIFICATION.md | accept→mitigate | _parse_frontmatter returns {} on non-`---` start; whole harvester wrapped — bad file skipped, not fatal |
| T-07-SC | Tampering | npm/pip installs | accept | No new dependencies added this plan (stdlib hashlib/re only); nothing to audit |
</threat_model>
