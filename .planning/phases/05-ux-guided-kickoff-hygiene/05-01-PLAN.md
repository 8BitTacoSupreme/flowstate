<objective>
Add a fast, scaffold-only `flowstate kickoff` command and enhance the shared interview so its
new questions appear in BOTH `init` and `kickoff` with zero divergence.

`kickoff` is the "scaffold and stop" entry point: it runs the (enhanced) interview, writes the
deterministic context files (PROJECT.md / ROADMAP.md / config.json / CLAUDE.md / starter fixture /

pack via `run_pack`, saves state, and EXITS — with NO `run_pipeline()`, NO bridge, NO LLM call.

Purpose: A new project can be scaffolded in seconds without spending tokens, then handed to GSD.
The enhanced interview is the single source of truth for both entry points (KICK-02).

all test-covered.
