<objective>
Make the compounding-eval report tell the truth about which mode produced it. Today `bench/report.py` hardcodes the cheap-mode `CAVEAT`, `mode_note`, and Rich table title, so a `--mode real` run falsely prints "cheap mode validates the apparatus, not causation." Thread the actual `mode`, `arm` (the `--layers` value), `sample_size` (K/trials), and which producer artifacts were present into `render_report` + `write_json`, and select mode-appropriate caveat/note/header text.

Purpose: A real-mode report currently masquerades as a cheap-mode regression guard — it makes a causal apparatus look like it made no causal claim. This is the first correctness gate of the "Make the Harness Real" milestone (HAR-01); v0.7.0/v0.8.0 benchmarking is blocked until reports are mode-honest.

Output: `bench/report.py` and `bench/compound_eval.py` updated (surgical threading only), plus a regression test proving real mode never leaks the cheap-mode string.
</objective>
