<objective>
Make the enforcement stage capable of failing and make the orchestrator + CLI surface that
failure honestly. Today `discipline.check_setup()` hardcodes `success=True` and the orchestrator's
Discipline step bypasses `_run_step` to mark the tool COMPLETED unconditionally ‚Äî so a repo passing
0/7 checks still reports "All steps succeeded."

Purpose: This is the honesty floor for the whole phase ‚Äî an audit that cannot fail cannot enforce.
Output: `check_setup` derives success from a required-set; Discipline flows through the existing
success‚ÜíCOMPLETED / failure‚ÜíBLOCKED runner; a new `flowstate discipline` command composes in CI.

Implements HON-01 (discipline can fail) and HON-02 (orchestrator surfaces it + CLI subcommand).
</objective>
