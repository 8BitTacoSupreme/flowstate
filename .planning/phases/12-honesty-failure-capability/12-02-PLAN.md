<objective>
Make the two bridge-backed adapters report failure honestly. Today `research.execute()` writes
"*Research failed*" notices into `report.md` and unconditionally returns `success=True`, and
`strategy.pressure_test()` passes the raw bridge result through — so an empty-but-`success=True`
bridge response is reported as a completed strategy. Also reconcile the `gsd_adapter` docstring,
which promises "optional LLM enrichment" the adapter never performs.

Purpose: honest per-adapter return values are what let `_run_step` mark these steps BLOCKED (the
runner already does that on `success=False`) — the foundation Phase 13's mechanisms build on.
Output: research fails when all topics fail; strategy fails on empty/failed output; the gsd
docstring matches the code.

Implements HON-03 (research surfaces failure), HON-04 (strategy surfaces failure), HON-06 (gsd docstring).
This plan touches only tools/ files — disjoint from plan 12-01, so it runs in parallel (wave 1).
</objective>
