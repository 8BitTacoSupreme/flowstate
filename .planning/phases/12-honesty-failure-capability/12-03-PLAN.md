<objective>
Make a live run with no `claude` CLI fail loud instead of faking success. Today `run_pipeline`
silently swaps the live bridge for a stub dry-run bridge (`orchestrator.py:171-173`) when the CLI is
missing; the adapters then write `[dry-run] claude prompt (N chars): ...` text into `report.md` /
`strategy.md` and report success — a run that produced no real research reports "All steps succeeded."

Purpose: closes the last of the three success-on-failure paths. With HON-03/HON-04 in place (plan
12-02), removing the swap lets the live bridge's own "claude CLI not found" failure propagate:
research/strategy return `success=False` → `_run_step` marks them BLOCKED, and no stub text is
written because the live bridge returns empty output, not the `[dry-run]` placeholder.

Output: the swap is removed; deterministic steps (context generation, discipline) still run; the
bridge-dependent steps BLOCK; genuine `--dry-run` is untouched.

Implements HON-05. Runs in wave 2: it edits orchestrator.py (also edited by plan 12-01) so it must
not run concurrently with 12-01, and it depends behaviorally on 12-02's HON-03 (research must return
success=False for the research step to BLOCK rather than fake success).
</objective>
