<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| installer -> project .claude/ | FlowState writes third-party files (incl. node_modules) into a user's project tree |
| vendored GSD node_modules -> install process | Copying (not executing) untrusted-origin JS at install time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-04 | Tampering | installer filesystem writes | mitigate | Assert every GSD destination resolves inside root/.claude (extend existing path-safety check); refuse writes outside GSD namespaces; skip source symlinks (reuse `_copy_tree` ignore) — no path traversal or symlink escape |
| T-15-05 | Elevation of Privilege | installing gsd-sdk + node_modules | mitigate | Copy as data (invoked via `node`, no exec bit needed); NEVER run any vendored file, npm, or postinstall during install — vendored code executes only when the user later invokes gsd-sdk |
| T-15-06 | Denial of Service | non-idempotent re-install | mitigate | dirs_exist_ok copy + idempotent `_register`; covered by a double-install test asserting no duplication |
| T-15-07 | Tampering | clobbering user-authored .claude content | mitigate | Overwrites scoped to GSD namespaces (.claude/skills/gsd-*, .claude/get-shit-done, the gsd-sdk dest); sibling user skills untouched |
</threat_model>
