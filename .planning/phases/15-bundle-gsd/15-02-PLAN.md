<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| installer → project .claude/ | FlowState writes third-party files into a user's project tree |
| vendored GSD → install process | Copying (not executing) untrusted-origin Node/JS at install time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-04 | Tampering | installer filesystem writes | mitigate | Assert every GSD destination resolves inside root/.claude (extend existing path-safety check); refuse writes outside GSD namespaces; skip source symlinks (reuse `_copy_tree` ignore) — no path traversal or symlink escape |
| T-15-05 | Elevation of Privilege | making gsd-sdk invokable | mitigate | Copy the CLI as data (+ exec bit only if install.js sets it); never run any vendored file during install — vendored code executes only when the user later invokes it |
| T-15-06 | Denial of Service | non-idempotent re-install | mitigate | dirs_exist_ok copy + idempotent `_register`; covered by a double-install test asserting no duplication |
| T-15-07 | Tampering | clobbering user-authored .claude content | mitigate | Overwrites scoped to GSD namespaces (.claude/skills/gsd-*, .claude/get-shit-done); sibling user skills untouched |
</threat_model>
