<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo tree → subprocess argv | Branch names, paths, and repo contents are attacker-influenceable input to git/pytest invocations (live path only) |
| hook file → auditor | `.git/hooks/pre-commit` contents are read/stat'd but must never be executed |
| dry-run flag → side-effect boundary | `--dry-run` must not cross into real subprocess execution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-06 | Elevation of Privilege | `_read_git_state` / `_run_project_tests` subprocess construction | mitigate | argv LIST only, never `shell=True`, never string interpolation of branch/path into a command; a malicious branch name is an inert argv element |
| T-13-07 | Elevation of Privilege | reading `.git/hooks/pre-commit` | mitigate | `_check_hook_contents` only `stat`s + reads size + checks `os.X_OK`; it NEVER executes the hook file |
| T-13-08 | Denial of Service | `_run_project_tests` running an arbitrary/long suite as a gating check | mitigate | bounded `timeout=_TEST_TIMEOUT` (900s); timeout -> None -> audit fails (honest fail, never a hang); tests stub the call; DRY-RUN skips the run entirely (zero spawns) |
| T-13-09 | Spoofing | git binary absent / not a repo / no test runner | mitigate | FileNotFoundError/CalledProcessError/TimeoutExpired degrade to None/False safe defaults; live `success` still derives honestly from `_REQUIRED_LIVE` (missing runner => tests_pass None => audit fails, not a fake pass) |
| T-13-10 | Tampering | dry-run spawning the target repo's real test suite / git (unwanted side effect) | mitigate | `check_setup(..., dry_run=True)` short-circuits before any `subprocess.run`; orchestrator threads `dry_run` into the discipline step; test asserts `subprocess.run.call_count == 0` under dry-run |
| T-13-SC | Tampering | npm/pip/cargo installs | accept | No package installs — stdlib `os`/`subprocess` + `git`/`pytest` already required by the project; no new runtime deps |
</threat_model>
