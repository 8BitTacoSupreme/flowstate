## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo tree â†’ subprocess argv | Branch names, paths, and repo contents are attacker-influenceable input to git/pytest invocations |
| hook file â†’ auditor | `.git/hooks/pre-commit` contents are read/stat'd but must never be executed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-06 | Elevation of Privilege | `_read_git_state` / `_run_project_tests` subprocess construction | mitigate | argv LIST only, never `shell=True`, never string interpolation of branch/path into a command; a malicious branch name is an inert argv element |
| T-13-07 | Elevation of Privilege | reading `.git/hooks/pre-commit` | mitigate | `_check_hook_contents` only `stat`s + reads size + checks `os.X_OK`; it NEVER executes the hook file |
| T-13-08 | Denial of Service | `_run_project_tests` running an arbitrary/long suite | mitigate | bounded `timeout` on the subprocess; timeout -> None (not-run), never hangs the audit; tests stub the call so no recursive suite spawns |
| T-13-09 | Spoofing | git binary absent / not a repo | mitigate | FileNotFoundError/CalledProcessError degrade to None/False safe defaults; `check_setup.success` still derives honestly from the required-set |
| T-13-SC | Tampering | npm/pip/cargo installs | accept | No package installs â€” stdlib `os`/`subprocess` + `git`/`pytest` already required by the project; no new runtime deps |
