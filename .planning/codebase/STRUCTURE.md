## Directory Purposes

**`flowstate/`:**
- Purpose: Main Python package for orchestration, adapters, and CLI
- Contains: Core orchestration logic, state management, tool adapters, event system, memory store
- Key files: cli.py (entrypoint), orchestrator.py (pipeline), state.py (Pydantic models), bridge.py (LLM integration)

**`flowstate/tools/`:**
- Purpose: Pluggable tool adapter implementations for each pipeline stage
- Contains: ToolAdapter base class, ResearchAdapter, StrategyAdapter, GSDAdapter
- Pattern: Each adapter implements execute() method; optional bridge and memory support; dry-run compatible

**`flowstate/events/`:**
- Purpose: Event-driven infrastructure for step coordination and memory persistence
- Contains: Event base class, EventBus, HandlerRegistry, @handler decorator
- Pattern: Pydantic-validated events; synchronous priority-ordered dispatch; error isolation

**`tests/`:**
- Purpose: Comprehensive test suite (155+ tests, 90% coverage target)
- Contains: Unit tests for CLI, state, orchestrator, interview, context, bridge, memory; integration tests for full pipeline
- Pattern: Uses conftest.py fixtures, Click CliRunner for CLI tests, monkeypatch for config isolation

**`research/` (generated):**
- Purpose: Output directory for research artifacts
- Contains: brief.md (research questions), report.md (findings from research adapter), strategy.md (from strategy adapter)
- Generated by: ResearchAdapter, StrategyAdapter during pipeline run

**`.planning/` (generated):**
- Purpose: Planning context for GSD and downstream tools
- Contains: PROJECT.md, ROADMAP.md, config.json (deterministically generated); codebase/ (from GSD skills)
- Generated by: Context generator, GSD adapter, later GSD skills

**`.claude/`:**
- Purpose: Claude Code tool configuration and project context
- Contains: CLAUDE.md (injected into all Claude sessions), settings.json (MCP server config)
- Created by: Context generator at init time; can be regenerated with `flowstate context`

## Key File Locations

**Entry Points:**
- `flowstate/__main__.py`: Invoked by `python -m flowstate`
- `flowstate/cli.py:main()`: Click group, entrypoint for `flowstate` CLI command
- `pyproject.toml:[project.scripts]`: Defines `flowstate = "flowstate.cli:main"`

**Configuration:**
- `pyproject.toml`: Project metadata, dependencies (click, pydantic, rich, sqlite-vec), test config (pytest, coverage 80%)
- `flowstate/config.py`: Global config persistence (~/.config/flowstate/config.toml); default root resolution
- `flowstate/state.py`: State file definition (flowstate.json); Pydantic schema with migration support

**Core Logic:**
- `flowstate/orchestrator.py`: run_pipeline() orchestrates 5 steps, run_phase() runs single GSD phase, print_status() renders status table
- `flowstate/bridge.py`: ClaudeBridge wraps `claude --print` with budget/model/effort overrides, allowed_tools scoping, max_turns bounding
- `flowstate/memory.py`: MemoryStore (SQLite FTS5), schema definition, search with BM25 ranking
- `flowstate/context.py`: Context generators (PROJECT.md, ROADMAP.md, CLAUDE.md, config.json)

**Tool Adapters:**
- `flowstate/tools/research.py`: ResearchAdapter splits topics, runs one bridge call per topic, merges into research/report.md
- `flowstate/tools/strategy.py`: StrategyAdapter runs single pressure-test call, writes research/strategy.md
- `flowstate/tools/gsd_adapter.py`: GSDAdapter writes .planning/ context files
- `flowstate/discipline.py`: Pure Python project audit (git, test config, hooks) — no LLM

**Testing:**
- `tests/conftest.py`: Shared fixtures (bus, state, memory_store, sample events)
- `tests/test_cli.py`: CLI command tests (version, help, init, status, launch, run, context, memory, check, fresh, config)
- `tests/test_orchestrator.py`: Pipeline sequencing, step runner, status table
- `tests/test_memory.py`: MemoryStore CRUD, FTS5 search, count by kind

## Naming Conventions

**Files:**
- Python modules: `snake_case.py` (e.g., orchestrator.py, memory.py, gsd_adapter.py)
- Test files: `test_<module>.py` (e.g., test_cli.py, test_orchestrator.py)
- Generated artifacts: `UPPERCASE.md` (e.g., PROJECT.md, ROADMAP.md, CLAUDE.md, ARCHITECTURE.md)
- Data files: `lowercase.json|.db` (e.g., flowstate.json, memory.db, config.json)

**Directories:**
- Packages: `snake_case/` with `__init__.py` (e.g., flowstate/, tools/, events/)
- Outputs: lowercase or UPPERCASE depending on tool (research/, .planning/, .claude/)
- Test directory: `tests/` at root level

**Classes:**
- Pattern: PascalCase, no "Manager"/"Handler" suffix unless semantically necessary
  - `FlowStateModel`, `MemoryStore`, `ToolAdapter`, `ClaudeBridge`, `EventBus`
- Enum: `MemoryKind`, `ToolStatus`, `EventPriority`
- Dataclass: `MemoryEntry`, `ToolResult`, `BridgeResult`, `BridgeConfig`, `AuditResult`

**Functions:**
- Pattern: `snake_case()`
  - Top-level: `run_pipeline()`, `run_interview()`, `launch_command()`, `write_context_files()`
  - Private: `_split_topics()`, `_build_topic_prompt()`, `_make_bridge()`, `_find_claude()`
- CLI commands: Underscores in function name become hyphens in CLI (e.g., `memory_search()` → `flowstate memory search`)

**Variables:**
- Constants: `UPPERCASE` (e.g., `TOOL_ORDER`, `STEP_LABELS`, `STEP_STYLES`, `SCHEMA_SQL`, `RESEARCH_SYSTEM_PROMPT`)
- Instance vars: `snake_case` (e.g., `self.root`, `self.dry_run`, `self._conn`, `self._keep_history`)
- Type hints: Used extensively (e.g., `root: Path | None`, `artifacts: list[str]`)

## Where to Add New Code

**New Feature (e.g., new tool in pipeline):**
- Primary code: `flowstate/tools/<tool_name>.py` — subclass ToolAdapter, implement execute()
- State tracking: Add to `FlowStateModel.tools` dict in `flowstate/state.py`
- CLI invocation: Add command to `flowstate/cli.py`
- Orchestrator: Add to `TOOL_ORDER`, `STEP_LABELS`, `STEP_STYLES` in `flowstate/orchestrator.py`
- Memory handlers: If needed, create handler in `flowstate/memory_handlers.py`
- Tests: `tests/test_<tool_name>.py`

**New Context File (e.g., custom brief for a tool):**
- Generator: Add function to `flowstate/context.py` (e.g., `generate_my_brief()`)
- Usage: Call from orchestrator during context write phase
- Downstream: Reference in tool adapter via `root / "path/to/file.md"`

**New CLI Command:**
- Location: Add @main.command() or @subgroup.command() in `flowstate/cli.py`
- Implementation: Typically calls orchestrator or state functions
- Help text: Use docstring for `--help` output

**New Memory Kind:**
- Enum: Add to `MemoryKind` in `flowstate/memory.py` (e.g., `EXPERIMENT = "experiment"`)
- Handler: If auto-storage needed, add logic to `flowstate/memory_handlers.py`
- Search: Automatically indexed by FTS5

**New Event Type:**
- Definition: Add class to `flowstate/events/event.py`, subclass Event, set event_type
- Emission: Call `bus.emit(MyEvent(...))` from orchestrator or adapter
- Handling: Register handler via `bus.on("my.event", handler)` or @handler decorator

**New Test:**
- Location: `tests/test_<module>.py`
- Pattern: Use fixtures from conftest.py (bus, state, memory_store, events)
- Isolation: Use tmp_path for file I/O, monkeypatch for config/env vars
- Coverage: Run `pytest --cov=flowstate` to check coverage

## Special Directories

**`flowstate/__pycache__/`:**
- Purpose: Python bytecode cache
- Generated: Automatically by Python interpreter
- Committed: No (in .gitignore)

**`tests/__pycache__/`:**
- Purpose: Python bytecode cache for tests
- Generated: Automatically by pytest
- Committed: No (in .gitignore)

**`htmlcov/`:**
- Purpose: HTML coverage report
- Generated: By pytest with `--cov-report=html`
- Committed: No (in .gitignore)
- Access: Open `htmlcov/index.html` in browser after test run

**`.git/`:**
- Purpose: Git repository metadata
- Generated: On first `git init`
- Committed: No (only local)

**`flowstate.egg-info/` or `.venv/`:**
- Purpose: Development install artifacts or virtual environment
- Generated: By `pip install -e .` or `python -m venv`
- Committed: No (in .gitignore)
